蜂群式无中心事件系统


公司主体：xHuman Ltd
通过一颗嵌入实体的机芯， 在事件发生时连接人和物， 让不相关的世界自然连在一起的系统。

第 0 章｜投资人阅读说明

本项目不是单一玩具产品，而是一个可长期运营、持续扩展的玩偶平台系统。

本轮投资用于推进既定系统方案，而非验证想法是否可行。

项目已完成公司注册、商标与专利布局，并具备完整产品与系统架构定义。

玩偶不是摆设：一个会“越来越聪明”的系统空间

本章节用于帮助投资人理解：蜂群玩偶平台不是一次性功能集合，而是一个具备长期演化空间的系统。

一、先澄清一个常见误解

蜂群玩偶平台中的玩偶，并不是展示品，也不是一次性互动电子玩具。

它们在系统中被定义为“角色节点”，其能力不会在出厂时被一次性写死。

二、“越来越聪明”真正指的是什么

这里所说的“越来越聪明”，并不是指玩偶本体在学习，也不是在玩偶中运行复杂 AI。

真正不断演化的是平台世界本身：

平台对历史事件、群体行为和长期模式的理解不断加深，从而下达更合理的行为决策。

换句话说：

玩偶本体保持简单，世界持续变得更聪明。

三、一个可以想象的演化路径

第一阶段：系统只能理解基础行为模式，例如某个玩偶被触发的频率。

第二阶段：系统开始理解行为之间的关联，不同玩偶逐渐表现出差异化反应。

第三阶段：系统能够判断整个世界所处的阶段，新加入的玩偶可以自然融入已有世界。

聪明的是“世界”，玩偶只是这个世界的身体。

四、为什么这种演化是可持续的

第一，不依赖硬件升级。系统能力提升来自平台规则和计算模型的升级，而非更换玩偶。

第二，不依赖持续在线。系统通过长期事件积累演化，而非实时高算力计算。

第三，规模越大反而越稳定。玩偶数量越多，世界的历史越完整，系统判断越可靠。

五、对投资价值的意义

蜂群玩偶平台的长期价值，不取决于某一代玩偶销量，而取决于这个世界积累了多少历史与规则。

这意味着平台具备“越用越有价值”的属性，而非卖完即结束。

第 1 章｜一句话讲清楚我们在做什么

蜂群玩偶平台，是一个让多个玩偶共同存在于同一个“世界”中的系统。

即使每个玩偶大多数时间处于休眠或离线状态，这个世界依然持续运转并不断演化。

第 2 章｜已被市场验证的现实背景（Labubu 案例）

以 Labubu 为代表的潮流玩偶，已实现数百万级销量，证明玩偶市场真实存在。

但现有潮流玩偶的商业模式主要是一次性销售：

  • 玩偶卖出后价值基本固定
  • 用户关系随时间减弱

蜂群玩偶平台并不否认这一成功，而是在此基础上进一步思考：

玩偶是否可以成为“持续存在的角色”，而不仅是被收藏的对象。

第 3 章｜核心创新：让玩偶“活着”的系统

蜂群玩偶的创新不在于更复杂的硬件，而在于系统思路的根本不同：

  1. 世界比单个玩偶更重要
  2. 玩偶无需持续在线
  3. 发生过的事情不会消失

结果是：

玩偶可以休眠，但世界持续存在；用户不是拥有一个玩偶，而是参与一个世界。

第 4 章｜和普通玩具 / AI 玩具的本质区别

普通玩具 / AI 玩具关注的是：“一个玩具有多聪明”。

蜂群玩偶平台关注的是：“一个世界能否长期存在并不断演化”。

普通玩具

  • 单个玩偶是主体
  • 依赖持续在线
  • 关机即停止

蜂群玩偶平台

  • 世界是主体
  • 玩偶可长期离线
  • 世界持续演化

这种差异不是功能升级，而是系统范式的差异，复制成本极高。

第 5 章｜创新如何被实现（非技术版）

  1. 玩偶的所有行为都会被记录
  2. 平台通过这些记录持续计算世界状态
  3. 玩偶只在需要时醒来执行简单指令

这使系统具备低功耗、高稳定性和长期可运营性。

6.3 平台核心 IP 的不可分割性（Non-separable Core Intellectual Property）

6.3.1 被定义为“核心 IP”的范围

核心 IP 包括但不限于：

  • 与上述内容直接相关的：架构文档、算法设计、协议定义、接口规范

📌 核心 IP 的定义基于“系统结构与规则”，而非具体代码实现。

6.3.2 不可分割原则（Non-separability）

  • 核心 IP 不得被单独授权、出售或拆分
  • 不得脱离整体系统语境，仅授权或转移其中某一模块
  • 不得将核心 IP 用于任何与蜂群玩偶平台构成直接或间接竞争关系的系统或产品

6.4 合作开发中产生的技术成果（Foreground IP）

6.4.1 默认归属原则

  • 系统级成果（包括架构设计、算法逻辑、平台机制）：→ 归属公司（xHuman Ltd）
  • 实现级成果（包括代码实现、配置文件、部署脚本）：→ 作为系统整体实现的一部分，整体归属公司
  • 合作方不因实施开发工作本身获得任何系统所有权或独立使用权

📌 上述归属原则，是“技术折股 / 技术入股”模式成立的前提条件。

6.4.2 合作方的合理权益

  • 通过股权形式参与并分享平台长期价值
  • 在协议明确约定的范围内：作为平台技术合作方对外展示合作案例
  • 在不构成竞争的前提下，复用其自身通用工程能力与经验

6.5 明确排除的权利（What Partners Do NOT Obtain）

  • ❌ 平台核心算法或系统的独立使用权
  • ❌ 将系统架构或模块拆解后单独商业化的权利
  • ❌ 在其他玩偶、陪伴系统或群体系统中复用本平台架构的权利
  • ❌ 对平台未来技术路线、版本演进或产品方向的否决权

6.6 技术折股场景下的 IP 处理（Critical）

折算的是：投入价值、风险共担，并非购买代码、系统或 IP 的使用权。

在该模式下：所有开发成果视为对公司的资本性投入；知识产权归属公司；合作方权利体现为股权，而非任何形式的 IP 切割或授权。

本条款用于明确区分：“战略型技术入股” ≠ “外包开发 + 隐性授权”。

  • 防止平台被模块化复制
  • 防止硬件或软件被单点绕开

蜂群玩偶平台的知识产权策略，并非为了最大化短期合作便利， 而是为了确保系统范式与平台价值在长期内不被拆解、复制或稀释。 合作方通过股权参与平台成长，而非通过拆分系统获取局部控制权。
